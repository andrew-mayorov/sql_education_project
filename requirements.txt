# SQL Internship Analytics Portfolio

Портфолио-проект для начинающего аналитика данных. Проект имитирует работу стажёра в маркетплейсе: есть SQLite-база с заказами, товарами, клиентами, сессиями, доставками, маркетинговыми расходами и обращениями в поддержку.

## Что показывает проект

Проект демонстрирует практические навыки SQL:

- `JOIN`, `GROUP BY`, `HAVING`, `CASE`;
- CTE через `WITH`;
- оконные функции `RANK`, `SUM OVER`;
- расчёт метрик: выручка, прибыль, средний чек, CAC, ROI, retention, SLA;
- когортный анализ, RFM-сегментация, ABC-анализ;
- работа с датами в SQLite.

## Список задач

1. Динамика выручки и среднего чека по месяцам.
2. Категории товаров: выручка, прибыль и маржинальность.
3. Топ-3 товара внутри каждой категории.
4. Повторные покупки и LTV покупателей.
5. RFM-сегментация клиентов.
6. Когортный анализ удержания.
7. Воронка конверсии по каналам привлечения.
8. Эффективность промокодов.
9. ROI маркетинговых каналов.
10. Анализ доставки и SLA.
11. Возвраты, отмены и обращения в поддержку.
12. ABC-анализ товаров по выручке.

## Как запустить

```bash
python -m venv .venv
source .venv/bin/activate  # для Windows: .venv\Scripts\activate
pip install -r requirements.txt
jupyter notebook
```

Откройте любую тетрадку из папки `notebooks/`. Все решения используют готовую базу `data/marketplace.sqlite`.

## Таблицы

- `customers` — клиенты, город, дата регистрации, канал привлечения;
- `products` — товары, категории, цена, себестоимость;
- `orders` — заказы, дата, статус, промокод, способ оплаты;
- `order_items` — товары внутри заказа;
- `payments` — платежи;
- `shipments` — доставки;
- `sessions` — сессии и шаги воронки;
- `marketing_spend` — расходы по каналам и месяцам;
- `support_tickets` — обращения в поддержку.
